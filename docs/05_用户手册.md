# 五、用户手册

## 5.1 系统概述

单细胞高维向量 ANN 检索系统是一个 Web 应用，帮助生物信息学研究者在数十万级别的单细胞数据中快速定位相似细胞，并通过可视化与条件过滤辅助分析。

## 5.2 部署与启动

### 5.2.1 一键启动（推荐）

```bash
git clone <repo-url>
cd ann_search
cp .env.example .env             # 修改 SECRET_KEY 等
make up                          # 启动全部服务
make migrate                     # 首次启动需要应用数据库迁移
```

启动完成后访问：

- 前端：<http://localhost:5173>
- API 文档：<http://localhost:8000/docs>

### 5.2.2 停止与清理

```bash
make down        # 停止
make logs        # 查看日志
make prune       # 移除卷 (会删除数据库！谨慎)
```

## 5.3 账号与登录

1. 打开前端首页，点击「注册」；
2. 填写用户名 (3~32 字符)、邮箱、密码 (≥ 8 位)；
3. 注册成功后自动跳转登录；
4. 登录后页面右上角可查看 / 修改个人信息、登出。

## 5.4 数据集管理

### 5.4.1 上传数据集

1. 进入「数据集」页面；
2. 点击「上传」按钮；
3. 选择本地 `.h5ad` 文件（建议 ≤ 2GB），可填写名称、描述；
4. 上传后系统会在后台异步解析：读取 `n_obs`、`n_vars`、`obs` 列、`obsm` 字段；
5. 解析完成后该数据集状态变为「就绪」。

### 5.4.2 数据集详情

- 元信息卡片：细胞数、基因数、obs 列数、可用 obsm；
- Tab：
  - **预览 obs**：前 100 行；
  - **UMAP 投影**：散点图，可按 obs 列着色；
  - **索引列表**：已构建 / 构建中 / 已卸载；
  - **检索入口**：跳转到检索页并预选该数据集。

### 5.4.3 删除数据集

- 在列表行或详情页点击「删除」，需二次确认；
- 关联索引、检索历史会一并删除。

## 5.5 索引构建

1. 在数据集详情页，点击「新建索引」；
2. 填写参数：
   - **算法**：Brute-force / FAISS-Flat / FAISS-IVF / HNSW；
   - **度量**：L2 / cosine / inner product；
   - **向量字段**：选择 `obsm` 中的可用键，默认 `X_pca`；
   - **算法参数**：IVF 的 `nlist` / `nprobe`，HNSW 的 `M` / `efConstruction` / `efSearch`。
3. 提交后任务进入后台队列，列表显示进度；
4. 状态变为「就绪」即可用于检索。

### 5.5.1 算法建议

| 数据规模 | 维度 | 推荐算法 | 备注 |
| --- | --- | --- | --- |
| < 1 万 | 任意 | Brute-force | 精确 |
| 1 ~ 10 万 | < 256 | HNSW | 召回高、查询快 |
| 10 ~ 100 万 | < 256 | HNSW / IVF | 视内存权衡 |
| > 100 万 | 任意 | IVF + PQ | 内存友好 |

## 5.6 检索查询

### 5.6.1 基本检索

1. 进入「检索」页面；
2. 选择数据集、索引；
3. 输入查询源（三选一）：
   - 已存在的细胞 ID；
   - 自定义向量（与索引维度一致）；
   - 上传一个小型 `.h5ad` / `.npy` 文件；
4. 设置 Top-K (默认 10)、距离度量；
5. 点击「检索」。

### 5.6.2 条件检索

在检索表单中展开「过滤条件」，按 obs 列添加条件：

- 等值：`cell_type = Hepatocyte`；
- 多值：`disease IN (NAFLD, Healthy)`；
- 范围（数值）：`AgeYears BETWEEN 0 AND 18`。

系统会先在符合条件的细胞子集中检索 Top-K。

### 5.6.3 多数据集联合检索（加分）

勾选多个数据集，系统会在每个数据集独立检索后按距离合并，返回全局 Top-K，并标记来源数据集。

### 5.6.4 自然语言检索（加分）

在搜索框直接输入自然语言（例如：「肝脏中表达 ALB 较高的肝细胞」），系统会：

1. 由 LLM 解析为「向量来源 + 过滤条件 + Top-K」；
2. 执行结构化检索；
3. 由 LLM 总结结果与细胞类型分布。

## 5.7 结果与可视化

- 结果表：列出 Top-K 细胞 ID、距离、关键 obs 列（cell_type、disease 等）；
- 散点图：UMAP 投影上将 Top-K 高亮，鼠标悬浮显示元信息；
- 距离直方图：辅助判断结果显著性；
- 「导出 CSV」按钮可下载结果。

## 5.8 历史与回放

- 「历史」页面列出最近的检索；
- 点击任意一条可回放参数与结果；
- 支持按数据集 / 索引 / 时间过滤。

## 5.9 常见问题 (FAQ)

**Q: 上传 h5ad 后一直处于「解析中」？**
A: 大文件解析较慢，请耐心等待；可在 `make logs` 中查看 worker 日志。如长时间无进展，请检查 Redis / Worker 是否正常运行。

**Q: 索引构建失败？**
A: 常见原因：所选 `obsm` 字段不存在、维度过低导致 IVF `nlist` 大于样本数、内存不足。请重新调整参数。

**Q: 检索结果与 Brute-force 不一致？**
A: ANN 是近似算法，召回率 < 100% 属正常。可在「索引详情」查看 Recall@10 评测值；可尝试增大 `efSearch` / `nprobe`。

**Q: 想要重置数据库？**
A: 执行 `make prune`（会删除所有数据），或仅删除 `pgdata` volume 后重新 `make up && make migrate`。

**Q: 修改环境变量后未生效？**
A: 修改 `.env` 后需执行 `make down && make up` 重新启动容器。
