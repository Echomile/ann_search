## 3.4 性能测试

### 3.4.1 ANN 引擎基准

| 指标 | 方法 | 目标 |
| --- | --- | --- |
| 召回率 Recall@10 | 与 Brute-force 对比 | HNSW ≥ 0.95，IVF ≥ 0.90 |
| 平均查询时延 | 1000 次随机查询 | HNSW ≤ 5ms，IVF ≤ 10ms |
| QPS | 并发 50 客户端 | ≥ 500 QPS |
| 索引构建时间 | 30 万 × 50 维 | HNSW ≤ 60s |
| 内存占用 | 加载后 RSS | ≤ 2GB |

### 3.4.2 API 压测

- 工具：`locust`，模拟 50 并发用户，持续 5 分钟；
- 场景：
  - 登录 + 检索（90%）；
  - 上传 + 索引构建（10%，异步立即返回）；
- 指标：P50 / P95 / P99 延迟、错误率（< 0.1%）、CPU / 内存峰值。

### 3.4.3 前端性能

- Lighthouse Performance ≥ 85；
- 首屏 ≤ 2s；
- 散点图 30 万点采用 WebGL 渲染，操作不卡顿。

## 3.5 缺陷管理

- 使用 GitHub Issues 跟踪缺陷，标签：`bug` / `enhancement` / `question`；
- 优先级：P0（阻塞）、P1（重要）、P2（一般）、P3（建议）；
- 每个缺陷需关联复现步骤、期望、实际、截图 / 日志。
