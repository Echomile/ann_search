# 一、项目概述

## 1.1 开发背景

随着单细胞测序技术 (Single-cell sequencing) 的成熟，一次实验可产生数十万甚至百万级别的细胞样本，每个细胞经表达矩阵处理与 PCA 等降维后形成数百维的高维向量。基于这些向量进行相似细胞检索是细胞类型注释、轨迹分析、跨数据集比较等任务的基础需求。

传统的精确最近邻 (kNN) 在大规模、高维数据上检索效率较低、内存占用高、响应时间长，难以满足交互式可视化场景。近似最近邻 (ANN, Approximate Nearest Neighbor) 算法（LSH、IVF、HNSW、PQ 等）通过牺牲少量召回率换取数量级的查询加速，已在图像检索、推荐系统中广泛使用，同样适用于单细胞数据检索。

本项目设计并实现一个面向单细胞高维向量数据的 Web 化 ANN 检索系统，提供数据上传、预处理、索引构建、检索查询、结果可视化与性能评测一体化能力。

## 1.2 项目目标

1. **核心检索**：支持上传 `.h5ad` 数据集，自动读取 `obsm["X_pca"]` 或自定义降维向量；提供 Top-K 相似细胞检索接口。
2. **多算法支持**：至少实现 Brute-force（baseline）、FAISS-IVF、HNSWLIB 三种索引，统一接口可切换。
3. **条件检索**：按 `obs` 元数据（如 `cell_type`、`disease`、`AgeGroup`）做过滤后的相似检索。
4. **多数据集管理**：支持数据集的增加、删除、查看，以及动态构建 / 加载 / 卸载索引。
5. **可视化**：提供 UMAP / t-SNE 二维投影、Top-K 检索结果高亮、性能指标看板。
6. **性能评估**：记录每次查询的耗时、返回数量、命中召回率 (与 Brute-force 对比) 等指标。
7. **加分功能**：
   - 多数据集联合检索；
   - 自研或改进的 ANN 算法；
   - RAG + LLM 自然语言检索接口。

## 1.4 可行性分析

### 1.4.1 技术可行性

- **数据规模**：单数据集 ~10 万细胞 × 50~200 维 PCA 向量，单精度浮点占用约 100MB，可全部驻留内存。FAISS 与 HNSWLIB 均支持百万级向量秒级查询。
- **算法成熟度**：FAISS、HNSWLIB 已在工业界广泛使用，开源稳定。scanpy 提供完整 h5ad 读取 / 降维 / 可视化。
- **Web 框架成熟度**：FastAPI 异步 + Uvicorn 支持高并发；React + Vite 提供极速开发体验。
- **团队能力**：成员具备 Python / 前端基础，参考资料完备，预计两周可完成 MVP。

### 1.4.2 资源可行性

- 单机部署即可（8GB 内存以上，无需 GPU）；
- 全部为开源软件，零授权费用；
- 数据集由课程提供（约 1.4GB liver h5ad）。

### 1.4.3 时间可行性

按项目计划，团队 3~5 人配合，可在 6~8 周内完成开发、测试、文档与展示。
